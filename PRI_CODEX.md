# PRI_CODEX.md

# WEFLOW 메뉴얼 기준 우선 수정 순서

이 문서는 회사에서 전달한 WEFLOW 메뉴얼 내용을 기준으로, 현재 프로젝트에서 먼저 맞춰야 할 수정 우선순위를 정리한 작업 기준서다.

## 현재 판단

- 현재 프로젝트는 홈, 서비스, 가격, 성공사례, 예약, 랜딩, 관리자 기능의 큰 골격은 구현되어 있다.
- 다만 회사 메뉴얼 기준으로 보면 문구, 버튼 이동, 레이아웃, 가격표, 예약 시간, 관리자 상세 UX, 우측 fixed quick menu 디자인이 아직 맞지 않는 부분이 있다.
- 특히 제출 전에는 "기능 존재 여부"보다 "메뉴얼과 같은 화면/흐름인지"가 중요하다.

## 우선 수정 순서

### 1. 헤더 메뉴 정리

- 메뉴얼 기준 네비게이션은 `홈, 서비스, 제작플랜&가격안내, 성공사례, 예약, 무료진단받기`이다.
- 현재 헤더 nav에는 `서비스`가 빠져 있고, `가격안내` 라벨도 메뉴얼의 `제작플랜&가격안내`와 다르다.
- 일반 사용자 화면에 `로그인`, `회원가입`이 노출되는 것이 제출물에 적절한지 결정해야 한다.
- `무료진단받기`는 메뉴 버튼 또는 CTA로 명확하게 보여야 한다.

### 2. 홈 히어로 영역 수정

- 작은 문구 `랜딩&홈페이지 제작 · 광고 운영 · 검색 상단 노출 · 맞춤형 웹 솔루션`이 실제 화면에 보여야 한다.
- 큰 문구 `문의로 이어지는 홈페이지를 만듭니다`는 2줄 기준으로 글자가 어색하게 끊기지 않게 조정한다.
- 버튼은 `무료 진단 신청`, `성공 사례 보기`, `WEFLOW 랜딩 페이지` 3개가 보여야 한다.
- `WEFLOW 랜딩 페이지` 버튼은 `/landing`으로 이동해야 한다.
- 작은 직사각형 태그 `케어 플랜`, `빠른제작`, `합리적 비용`을 히어로 영역에 표시한다.

### 3. 홈 성공사례/프로세스/후기 보강

- 홈 성공사례 영역은 메뉴얼처럼 왼쪽 설명 박스와 오른쪽 성공사례 이미지 5개 구성을 맞춘다.
- `살펴보기 ->` 버튼은 메뉴얼 기준 무료진단창으로 연결되어야 한다.
- 홈 제작진행과정은 `제작진행과정 4칸`과 `6단계 제작 프로세스 6칸`을 양옆으로 나란히 배치한다.
- 후기 문구는 메뉴얼에 적힌 업종별 후기 형식과 최대한 맞춘다.
- `후기 더보기 ->` 버튼을 노출하고, 클릭 시 문의창 또는 무료진단창으로 연결한다.

### 4. 가격표를 메뉴얼 원문 기준으로 재정리

- 제작 플랜은 원가와 할인가를 2줄로 보여주고, 원가는 취소선 또는 할인 전 가격처럼 표현한다.
- START 랜딩페이지: `498,000원 -> 249,000원`
- GROW 홈페이지: `1,980,000원 -> 990,000원`
- MASTER 프리미엄: `2,980,000원 -> 1,490,000원`
- MASTER 프리미엄은 색을 다르게 하고 왕관 표시로 강조한다.
- 케어 플랜도 원가와 할인가를 같이 표시한다.
- WE CARE: `월 170,000원 -> 월 89,000원~`
- FLOW CARE: `월 378,000원~ -> 월 189,000원~`
- WEFLOW CARE: `월 678,000원~ -> 월 339,000원~`
- WEFLOW CARE는 색을 다르게 하고 왕관 표시로 강조한다.
- 광고 플랜도 메뉴얼 기준의 체크리스트와 원가/할인가를 반영한다.
- 네이버 광고: `298,000원 -> 149,000원~`
- 당근 플레이스 광고: `158,000원 -> 79,000원~`
- 각 가격 아래 `VAT 포함` 문구를 명확히 표시한다.
- 도메인, 광고비, 유지보수, 페이지 추가 비용 안내 문구를 가격 페이지 하단에 정리한다.

### 5. 예약 페이지 요구사항 반영

- 예약 시간대는 `09:00`부터 `18:30`까지 30분 간격으로 총 20개만 표시한다.
- 시간 버튼은 가로 5개, 세로 4줄 형태로 맞춘다.
- 현재 시간이 13시라면 13시 이전 시간은 선택 불가능해야 한다.
- 직접 입력 시간대 입력칸은 유지한다.
- 제작종류 옵션은 `랜딩페이지 제작`, `홈페이지 제작`, `랜딩&홈페이지 제작`, `기타(weflow 케어플랜)` 형태로 맞춘다.
- 예약 제출 데이터는 관리자 대시보드에 즉시 반영되어야 한다.

### 6. 관리자 대시보드 상세 UX 보강

- 예약 관리와 문의 관리 모두 엑셀 다운로드 기능은 유지한다.
- 상태, 이름, 연락처, 접수일, 희망 일정, 관리 컬럼을 메뉴얼 기준으로 정리한다.
- 관리 영역에는 `완료`, `진행중`, `삭제` 버튼 요구가 있다.
- 예약도 `진행중` 상태를 둘지 결정하고, 둘 경우 서버 상태값과 UI를 같이 수정한다.
- 관리 옆 아래화살표를 추가한다.
- 아래화살표 클릭 시 제작 종류, 업종, 추가요청사항 같은 상세 내용이 펼쳐져야 한다.
- 상태 변경, 삭제는 클릭 즉시 실시간 반영되어야 한다.

### 7. 랜딩페이지 메뉴얼 기준 정리

- 랜딩페이지 큰 문구는 `문의로 이어지는 홈페이지를 만듭니다`.
- 작은 문구는 `기획부터 제작, 광고 연동, 운영 관리까지 WEFLOW가 함께합니다.`로 유지한다.
- `무료 진단 후 견적받기`는 문의창 또는 무료진단창으로 연결한다.
- 메뉴얼에 따르면 `실제 제작 성공 보기`도 문의창으로 연결하라고 되어 있으므로, 현재 `/cases` 이동이 맞는지 다시 결정한다.
- 오른쪽 sticky 문의창은 관리자 대시보드 문의 목록에 저장되어야 한다.
- 랜딩 혜택 섹션은 `WEFLOW CARE PLAN`, `합리적인 비용`, `24시간 상담 가능`, `제작 후 운영 관리`, `광고 연동 지원` 문구가 실제 화면에 정확히 보여야 한다.
- 현재 랜딩 화면이 홈의 혜택 데이터를 재사용하는 부분이 있으므로, 메뉴얼 전용 문구가 보이도록 정리한다.
- 가격 카드, 서비스 제작진행과정, 후기창은 메뉴얼 기준으로 재사용한다.

### 8. 금지/주의 문구 점검

- 메뉴얼에 `병원 의료기술 절대 넣지 마시오`라고 되어 있다.
- 코드, placeholder, 문구, 사례, 이미지 alt 등에 병원/의료기술 관련 표현이 남아 있는지 점검한다.
- 현재 업종 placeholder에 `병원` 예시가 있으면 제거하는 편이 안전하다.

### 9. 우측 fixed quick menu 디자인 전면 수정

- 현재 우측 `카톡상담/전화상담/무료상담신청/Top/마이페이지` 형태의 세로 아이콘 메뉴를 제거하거나 축소해야 한다.
- 회사가 현재 우측 fixed 버튼 묶음을 선호하지 않으므로, 스크린샷처럼 카드형 fixed 상담 박스로 변경한다.
- PC에서도 스크롤 시 계속 따라오되, 버튼은 `무료 진단 신청`, `카카오톡 상담`, `전화 문의` 중심으로 정리한다.
- 하단에는 `문의 바로가기 / 진단 · 카카오 · 전화` 같은 작은 보조 플로팅 카드도 추가 검토한다.
- 디자인은 Tailwind로 구현해도 되고, 필요하면 Ant Design의 `FloatButton`, `Card`, `Button` 컴포넌트를 사용해도 된다.
- 수정 대상 후보:
  - `src/components/layout/FloatingQuickMenu.jsx`
  - `src/data/commonText.js`
  - `src/app/globals.css`

## 검수 체크리스트

- `무료 진단 신청`, `무료진단 후 견적받기`, `더보기`, `자세히보기` 같은 CTA가 메뉴얼의 의도대로 이동하는지 확인한다.
- 예약/무료진단 제출 후 관리자 페이지에 데이터가 보이는지 확인한다.
- 관리자 상태 변경/삭제가 실제 JSON 저장소에 반영되는지 확인한다.
- 모바일에서 글자, 이미지, 버튼이 잘리지 않는지 확인한다.
- 우측 fixed 상담 카드와 하단 4개 고정바가 서로 겹치지 않는지 확인한다.
- `yarn lint`와 `yarn build`를 통과시킨다.
