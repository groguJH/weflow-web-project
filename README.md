## 프로젝트 개요

회사에서 제공한 기존 프로젝트를 바탕으로, WEFLOW 서비스 소개 페이지를 실제 문의와 예약으로 이어지는 구조로 확장했습니다.

단순히 화면을 꾸미는 것보다, 사용자가 서비스를 이해하고 바로 상담 신청을 할 수 있도록 다음 흐름을 중심으로 작업했습니다.

목적

- 홈에서 서비스 장점 확인
- 가격과 제작 과정을 확인
- 성공사례를 보고 신뢰 형성
- 무료진단 또는 예약 신청
- 관리자 페이지 생성 및 CRUD관리, 접수 데이터 확인 및 처리

## 실행 방법

패키지 설치:

```bash
yarn install
```

개발 서버 실행:

```bash
yarn dev
```

빌드:

```bash
yarn build
```

린트:

```bash
yarn lint
```

## 관리자 계정

과제 환경에서는 별도 DB 대신 기존 프로젝트 그대로 로컬 JSON 데이터를 사용했습니다.

```txt
ID: admin
PW: weflow1234
```

관리자 페이지:

- `/admin`
- `/admin/dashboard`
- `/admin/cases`

## 디자인 및 UX 개선 Before / After

회사에서 제공한 기존 프로젝트는 기본적인 페이지 구조와 콘텐츠가 있었지만, 과제 요구사항을 기준으로 보면 전환 흐름, 반응형 비율, 관리자 기능, 실제 데이터 저장 측면에서 보완이 필요했습니다.

### 전체 디자인 방향

| 구분        | Before                                                 | After                                                                    |
| ----------- | ------------------------------------------------------ | ------------------------------------------------------------------------ |
| 화면 구성   | 정적인 섹션 중심 구성                                  | 문의 전환을 기준으로 Hero, CTA, 혜택, 사례, 예약 흐름 재구성             |

| CTA         | 버튼 스타일과 위치가 섹션마다 다소 분산                | 무료진단, 사례보기, 서비스 더 알아보기 등 CTA 스타일과 목적 정리         |
| 인터랙션    | 정적 UI 중심                                           | Framer Motion, 캐러셀, 스크롤 reveal, hover 상태를 추가                  |

## 라이브러리 활용 방식

요구사항을 구현할 때 단순히 라이브러리를 추가하는 것이 아니라, 각 라이브러리가 필요한 역할에 맞게 사용되도록 정리했습니다.

| 라이브러리     | 사용 위치                                  | 사용 이유                                                                           |
| -------------- | ------------------------------------------ | ----------------------------------------------------------------------------------- |
| Framer Motion  | 홈 Hero, 오브젝트 등장 애니메이션          | 첫 진입 시 문구와 로고 영역을 순차적으로 보여주기 위해 사용                         |
| Ant Design     | 관리자 로그인, 대시보드, 테이블, 모달, 폼  | 관리자 페이지는 기능 중심 화면이라 안정적인 UI 컴포넌트를 빠르게 구성하기 위해 사용 |
| react-scroll   | 우측 fixed Top 버튼, 스크롤 이동           | Top 버튼 클릭 시 부드럽게 상단으로 이동하기 위해 사용                               |
| lucide-react   | 네비게이션, fixed 버튼, 관리자 버튼 아이콘 | 아이콘 스타일을 통일하고 커스텀 SVG 작성을 줄이기 위해 사용                         |
| Tailwind CSS 4 | 전체 레이아웃 및 반응형 스타일             | 기존 프로젝트 톤을 유지하면서 빠르게 섹션 단위 스타일을 조정하기 위해 사용          |

### Framer Motion 사용 방식

Hero 화면은 페이지에 들어왔을 때 한 번만 순차적으로 등장하도록 구성했습니다.

- 배경과 레이아웃은 먼저 유지
- 메인 문구 fade-in
- 설명과 CTA fade-in
- 우측 WEFLOW 로고와 원형 배지 fade-in
- 사용자가 다시 방문했을 때 매번 반복되지 않도록 세션 기준 처리

### Ant Design 사용 방식

관리자 페이지는 일반 사용자 페이지와 다르게 데이터 확인과 조작이 핵심입니다.  
그래서 직접 테이블과 모달을 모두 만드는 대신 Ant Design을 사용했습니다.

- 로그인 Form
- Alert
- Dashboard Card
- Statistic
- Tabs
- Table
- Modal
- Popconfirm
- Button

Ant Design은 관리자 영역에서만 사용하고, 사용자 페이지의 디자인 톤은 기존 Tailwind 기반 스타일을 유지했습니다.

### react-scroll 사용 방식

우측 fixed 메뉴의 `Top` 버튼은 현재 페이지 최상단으로 자연스럽게 이동해야 했습니다.  
이를 위해 `react-scroll`의 `animateScroll`을 사용했습니다.

### 데이터 저장 방식

과제 환경에서 실제 DB 연결은 부담이 크기 때문에, 로컬 JSON을 DB처럼 사용하는 방식으로 구현했습니다.

- 문의 저장: `/api/inquiries`
- 예약 저장: `/api/reservations`
- 관리자 조회: `/api/admin/records`
- 관리자 상태 변경/삭제: `/api/admin/*`
- 저장 파일: `src/server/adminDb.json`

## 라이브러리 및 코드 구조

프로젝트는 기능이 많아지면서 한 파일에 로직을 몰아넣기보다, 페이지, 섹션, 공통 UI, 데이터, 서버 로직을 역할별로 나눴습니다.

### 전체 폴더 구조

```txt
src
├─ app
│  ├─ page.js
│  ├─ landing
│  ├─ services
│  ├─ pricing
│  ├─ cases
│  ├─ reservation
│  ├─ privacy
│  ├─ terms
│  ├─ admin
│  └─ api
├─ components
│  ├─ layout
│  ├─ ui
│  └─ admin
├─ data
│  ├─ metadata.js
│  ├─ commonText.js
│  ├─ pricingText.js
│  └─ casesText.js
├─ features
│  ├─ home_page
│  ├─ landing_page
│  ├─ services_page
│  ├─ pricing_page
│  ├─ cases_page
│  ├─ reservation_page
│  ├─ legal_page
│  └─ admin
└─ server
```

### 구조를 나눈 기준

| 폴더 | 역할 | 사용 이유 |
| --- | --- | --- |
| `src/app` | Next.js App Router 라우트 | 페이지 URL, API Route, metadata 관리 |
| `src/components/layout` | Header, Footer, fixed menu | 모든 페이지에서 공유되는 레이아웃 분리 |
| `src/components/ui` | 버튼, 모달, 폼 필드, 섹션 헤더 등 | 여러 페이지에서 반복되는 UI 재사용 |
| `src/components/admin` | 관리자 전용 Provider | Ant Design 설정을 관리자 영역에만 적용 |
| `src/data` | 화면 문구, 가격, 사례, 약관 데이터 | UI와 데이터를 분리해 수정이 쉽도록 구성 |
| `src/features/*` | 페이지별 섹션 컴포넌트 | 홈, 가격, 예약, 관리자 등 도메인별 모듈화 |
| `src/server` | JSON 저장소, 관리자 인증, 데이터 처리 | API Route에서 공통으로 쓰는 서버 로직 분리 |

### 메타데이터 구조

Next.js App Router의 `metadata` export를 사용해 페이지별 SEO 정보를 정리했습니다.

```txt
src/data/metadata.js
→ 공통 사이트명, 기본 설명, 공유 이미지, metadata 생성 함수 관리
→ src/app/layout.js에서 전체 기본 metadata 설정
→ 각 page.js에서 페이지 목적에 맞는 metadata 설정
```

이 구조를 사용한 이유:

- 사이트 전체 기본값과 페이지별 정보를 분리
- title template을 통해 `페이지명 | WEFLOW` 형식 통일
- canonical 경로를 페이지별로 명확히 지정
- 관리자/회원가입 준비 페이지는 검색 노출을 막기 위해 `noIndex` 처리
- favicon과 공유 이미지를 한 곳에서 관리해 변경이 쉽도록 구성

| `Tailwind CSS` | 전체 컴포넌트 | 반응형, 색상, 간격, 레이아웃 스타일링 |

### 데이터 흐름 구조

문의와 예약은 클라이언트 폼에서 API Route로 전달되고, API Route는 서버 저장소 함수를 통해 JSON 파일에 기록합니다.

```txt
사용자 폼
→ /api/inquiries 또는 /api/reservations
→ src/server/adminStore.js
→ src/server/adminDb.json
→ /admin/dashboard에서 조회 및 관리
```

성공사례는 기본 데이터와 관리자 수정 데이터를 함께 고려하도록 구성했습니다.

```txt
src/data/casesText.js 기본 사례 데이터
→ src/server/adminStore.js에서 정규화
→ /cases 공개 페이지 출력
→ /admin/cases에서 수정
→ src/server/adminDb.json에 저장
```

### 관리자 기능 구조

관리자 기능은 사용자 페이지와 분리해 `features/admin` 아래에 모았습니다.

```txt
features/admin
├─ components
│  ├─ AdminLoginForm.jsx
│  ├─ AdminDashboardClient.jsx
│  ├─ AdminRecordsTable.jsx
│  ├─ AdminExportButtons.jsx
│  └─ AdminCasesClient.jsx
└─ utils
   └─ exportAdminExcel.js
```

이 구조를 사용한 이유:

- 관리자 기능이 사용자 페이지와 섞이지 않도록 분리
- 예약/문의 테이블과 모바일 카드 UI를 같은 컴포넌트에서 관리
- 엑셀 다운로드 로직은 UI와 분리해 `utils`로 관리
- Ant Design은 관리자 영역에서만 사용해 사용자 페이지의 디자인 톤과 분리

이후 주요 레이아웃은 rem 단위와 미디어쿼리를 함께 사용해 정리했습니다.

추가로 다음 부분도 반응형에 맞게 조정했습니다.

- 홈 주요 섹션 높이와 간격

**회사 요구사항**

따라서 검색 결과, 브라우저 탭, 링크 공유 미리보기에서 서비스 성격이 명확히 보여야 했습니다.

필요하다고 판단한 부분:

- 각 페이지의 목적에 맞는 title과 description
- 검색엔진이 인식할 canonical 경로
- 카카오톡, 블로그, SNS 공유 시 보이는 Open Graph 정보
- 브라우저 탭에 보이는 favicon
- 관리자 페이지처럼 외부 노출이 필요 없는 페이지의 검색 제외

**이행 방식**

- `src/app/layout.js`에 사이트 전체 기본 metadata 설정
- `metadataBase`로 절대 URL 기준 지정
- `title.template`으로 페이지 제목 형식 통일
- `keywords`, `authors`, `creator`, `publisher` 추가
- Open Graph와 Twitter Card 설정
- `public/main_icon.png`를 공유 이미지로 연결
- `public/icon.png`를 favicon, shortcut icon, apple icon으로 연결
- `/`, `/landing`, `/pricing`, `/cases`, `/reservation`, `/services`, `/privacy`, `/terms`에 페이지별 metadata 추가
- `/admin`, `/admin/dashboard`, `/admin/cases`, `/signup`은 `noIndex`로 검색 노출 방지

이렇게 구성해 단순히 화면만 완성한 것이 아니라, 회사가 실제 서비스처럼 보여줄 때 필요한 SEO 기본값과 브랜딩 요소까지 정리했습니다.

- `yarn build` 통과 확인

## 주요 라우트

- `/` 홈
- `/landing` 랜딩페이지
- `/services` 서비스 제작 과정
- `/pricing` 제작플랜 및 가격 안내
- `/cases` 성공사례
- `/reservation` 상담 예약
- `/privacy` 개인정보처리방침
- `/terms` 이용약관
- `/admin` 관리자 로그인
- `/admin/dashboard` 관리자 대시보드
- `/admin/cases` 성공사례 수정

## 기술 스택

- Next.js 16.2.6 App Router
- React 19.2.4
- Tailwind CSS 4
- Framer Motion
- Ant Design
- Lucide React
- react-scroll

## 검증 결과

아래 명령어 기준으로 정상 통과를 확인했습니다.

```bash
yarn lint
yarn build
```

## 참고 사항

- 과제 환경상 별도 DB 대신 로컬 JSON 파일을 사용했습니다.
- 관리자 데이터는 `src/server/adminDb.json`에서 관리됩니다.
- Next.js 빌드는 한글 경로 환경에서 Turbopack 문제가 발생할 수 있어 `--webpack` 옵션으로 실행되도록 설정했습니다.
- `next.config.mjs`에 `outputFileTracingRoot`를 지정해 workspace root 경고를 정리했습니다.
